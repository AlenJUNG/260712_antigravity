# Android Studio 설치 · 셋업 가이드 (Windows 11)

> 숲나들e 알림 **안드로이드 앱(Kotlin + Jetpack Compose)** 개발 환경을 처음부터 준비하는 초보자용
> 단계별 가이드. 앱 코딩 세션 착수 전, 이 문서대로 환경만 갖춰두면 된다.
>
> - 앱 UI는 화면 프로토타입으로 흐름 확정 완료(별도 아티팩트). 이 문서는 그 화면들을 실제 Compose
>   코드로 옮겨 **에뮬레이터에서 로컬 백엔드에 붙여 눈으로 테스트**하기 위한 개발 환경 준비다.
> - 백엔드는 이미 구현 완료(`02_구현/src/`). 앱은 그 REST API를 호출하는 얇은 클라이언트다.

---

## 0. 사전 확인 — 내 PC가 되는지

| 항목 | 필요 | 확인법 |
|---|---|---|
| OS | Windows 11 64bit | ✅ 충족 |
| RAM | 최소 8GB, **권장 16GB+** | 에뮬레이터가 무겁다 |
| 디스크 | **8GB 이상 여유**(SSD 권장) | Studio + SDK + 에뮬 이미지 |
| CPU 가상화 | VT-x(Intel)/SVM(AMD) 켜짐 | 작업관리자 → 성능 → CPU → "가상화: 사용" |
| JDK | **별도 설치 불필요** | Android Studio에 JBR(JDK) 내장 |

> 가상화가 "사용 안 함"이면: 재부팅 → BIOS/UEFI 진입 → Intel VT-x 또는 AMD SVM 활성화.

---

## 1. 다운로드

둘 중 하나:
- **웹**: <https://developer.android.com/studio> → "Download Android Studio" (약 1GB)
- **PowerShell**: `winget install -e --id Google.AndroidStudio`

---

## 2. 설치

1. 받은 `.exe` 실행 → **Next**
2. 컴포넌트 선택에서 **Android Studio** + **Android Virtual Device** 둘 다 체크
3. 설치 경로 기본값 유지 → **Install** → 완료 후 **Start Android Studio**

---

## 3. 첫 실행 마법사 (Setup Wizard)

1. *Import settings* → **Do not import settings**
2. 데이터 공유 → 원하는 대로
3. Install Type → **Standard** (초보자 권장)
4. 테마 선택
5. **SDK Components** 확인 → 그대로 **Finish**
6. Android SDK·플랫폼·에뮬레이터 이미지 **자동 다운로드**(수 GB, 10~30분). 라이선스는 **Accept**.

---

## 4. 가상화 기능 켜기 (에뮬레이터 필수)

Windows 11은 **Windows Hypervisor Platform(WHPX)** 을 쓴다:
1. 시작 → "**Windows 기능 켜기/끄기**" 검색
2. **Windows Hypervisor Platform** 체크 (있으면 **가상 머신 플랫폼**도)
3. 확인 → **재부팅**

> 안 켜면 에뮬레이터가 "hardware acceleration not available" 오류로 안 켜진다.
> (구형 Intel HAXM은 폐기됨. AMD는 WHPX 또는 AEHD 사용.)

---

## 5. 가상 기기(에뮬레이터) 만들기

1. 시작 화면 → **More Actions** → **Virtual Device Manager**
2. **Create Device** → Phone → **Pixel 7**(또는 8) → Next
3. System Image → 최신 API(예: **API 35, x86_64**) 옆 **Download** → 선택 → Next
4. 이름 확인 → **Finish**
5. 목록에서 **▶(Play)** 로 부팅 테스트 (첫 부팅 1~2분)

---

## 6. 새 프로젝트 만들기 *(앱 코딩 세션에서 실제 수행 — 미리 익혀두기)*

1. **New Project** → **Empty Activity**(Compose) → Next
2. 설정:
   - Name: `숲나들e 알림`
   - Package name: 예) `kr.foresttrip.alert`
   - Language: **Kotlin**
   - Minimum SDK: **API 26 (Android 8.0)**
   - Build config: **Kotlin DSL**
3. **Finish** → 첫 **Gradle sync**(최초 몇 분, 인터넷 필요)
4. 상단 기기 드롭다운에서 에뮬레이터 선택 → **▶ Run** → "Hello Android"가 뜨면 환경 완성

---

## 7. 우리 백엔드에 붙이기 (핵심)

앱을 로컬 백엔드에 연결해 눈으로 테스트할 때 두 가지만 기억하면 된다:

- **주소**: 에뮬레이터에서 내 PC의 `localhost` = **`10.0.2.2`**.
  백엔드가 `localhost:3000`이면 앱에서는 **`http://10.0.2.2:3000`** 으로 호출.
  (실기기 USB면 같은 WiFi + PC의 LAN IP `192.168.x.x`.)
- **http 허용(개발용)**: TLS 없이 붙으려면 `AndroidManifest.xml`의 `<application>`에
  `android:usesCleartextTraffic="true"`. **배포 시 제거** — 운영은 Cloudflare 터널 HTTPS 사용.
- **인증**: 모든 `/v1/*` 요청에 `Authorization: Bearer <APP_TOKEN>` 헤더 필수(ADR-0005).
  로컬 테스트 토큰은 `02_구현/.env`의 `APP_TOKEN`.

백엔드 기동·엔드포인트 테스트는 `docs/테스트가이드.md` 참조.

## 8. 자주 겪는 문제 (트러블슈팅)

| 증상 | 원인 · 해결 |
|---|---|
| 에뮬레이터가 안 켜짐 | 4단계 WHPX 미설정 또는 BIOS 가상화 꺼짐 |
| Gradle sync 실패 | 인터넷/프록시 확인, `File → Sync Project with Gradle Files` 재시도 |
| 앱에서 백엔드 연결 안 됨 | `localhost`가 아니라 **`10.0.2.2`** 를 썼는지, 백엔드가 켜져 있는지, `usesCleartextTraffic` |
| 401 Unauthorized | `Bearer <APP_TOKEN>` 헤더 누락/불일치 |
| 에뮬레이터가 느림 | RAM 부족 → AVD 설정에서 RAM 축소, 또는 실기기 USB 디버깅 사용 |
