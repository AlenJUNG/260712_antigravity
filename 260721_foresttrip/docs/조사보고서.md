- 조사일: 2026-07-21
- 방법: 공공데이터포털 조사 + 사이트 원본 HTML/JS 분석 + curl/Playwright 실호출

| 엔드포인트                                | 방법                 | 결과                                                 |

### 3.1 NetFunnel 게이트
- `netfunnel.js`(STCLab)의 `.wseq` 프로토콜, 혼잡 시 `Server=503` 큐.
- 서버가 유효한 `netfunnel_key` 를 요구 → curl 빈 키는 무조건 차단.
- Playwright로 실제 페이지를 띄우면 `fn_top_goSearch()` 호출 시 키가 정상 발급되어 통과./
- 검증: 강원 2026-08-19(평일) → 31곳 중 **예약가능 24 / 예약불가 7** 정확 반환.
  2026-08-01(토, 추첨일) → 예약가능 0 (선착순 조회 기준 정상).

### 3.2 결과 항목 DOM
```
.rc_item
  .rc_ti i   → "[예약가능]" / "[예약불가]"
  .rc_ti b   → "[공립](홍천군)가리산자연휴양림"
  .stay_info → "[객실] 39개 / [야영장] 0개", "[최대] 338명"
```

## 4. 개관연도 / 신축여부 조사

- 표준데이터·예약 API에는 개장연도 필드 **없음**.
- **산림청 국립자연휴양림관리소 '일반현황'** 페이지에 국립 47곳의
  **지정/조성/개장 연도 표** 존재 → 수집 가능.
  - 수집 결과: 47곳(유명산 1989 … 금산 2025). `data/national-open-years.json`.
- 공립·사립: 정형 통합 데이터 없음 → `null`.
- **신축여부**: 공식 필드 없음 → "개장 N년 이내"로 근사(국립만 신뢰).
  객실동 단위 신축은 공지사항 비정형 텍스트로만 확인 가능(자동화 부적합).

### 4.1 이름 매칭
- 예약 API "(강릉시)대관령자연휴양림" ↔ 일반현황 "대관령"
  → `(시군)` 접두 + `자연휴양림/휴양림` 접미 제거 후 비교.
- 강원 국립 14곳 중 12곳 자동 매칭. 예외:
  - `화천숲속야영장` = 표의 `화천야영장` → 별칭 보정.
  - `진부령` = 국립관리소 표에 없음(지방청 국유림) → null 유지.

## 5. 로그인 게이트 (객실 단위)

- `fcfsRsrvtPssblGoodsDetls.do` 는 비로그인 시 로그인 페이지를 반환(실측).
- 따라서 `숙박시설`/`시설상세`/`대기가능` 은 **로그인 세션이 있어야** 조회 가능.
- 로그인 폼 구조(실측):
  - `POST /com/login`, 필드 `loginId`(#mmberId) · `loginPwd`(#gnrlMmberPssrd) · `_csrf` · `targetUrl`
  - **캡차 없음, 클라이언트 암호화 없음, 비밀번호 5회 오류 시 계정 잠금**
- Playwright로 폼 입력·제출하면 로그인 성공 → 같은 컨텍스트로 객실 상세 접근 가능.

### 5.1 예약(01) vs 대기(02) 모드 — 대기 물량은 별도 조회
- 객실 검색 기본은 `rsrvtWtngSctin=01`(예약가능 목록) → **추첨/만석 숙박은 안 보임**.
  주말·공휴일 전날(예 8/1 토)은 숙박이 추첨이라 예약 모드에선 캠핑(야영데크)만 뜬다.
- 대기가능 물량은 goods 페이지의 **`fn_wtng_goSrch()`(=`rsrvtWtngSctin=02`)** 로 재조회해야 나온다.
  → 이걸로 **숲속의집·휴양관 등 숙박 대기물량**이 대거 포착됨(8/1 전국 대기 1,000+건).
- **대기신청은 숙박일 2일 전까지만 가능**(`selectGoodsWtngAppliPssblDtYn.do`).
- **대기 순번**은 대기 모드 `.list_box .opt4`("2순위"/"3순위")에 있음.

### 5.2 세션 재사용
- 로그인 후 `storageState`(세션쿠키)를 `auth.json`으로 저장하면, 이후 조회는 재로그인 없이
  세션 재사용으로 가능(비밀번호 입력 아님). 병렬(동시 6요청)에도 NetFunnel 세션오류 0 확인.

## 6. 합법성 / 유의사항

- robots.txt: 예약/조회 경로는 차단하지 않음(검색·FAQ 일부만 Disallow).
- 본 서비스는 **조회·안내**까지. 자동 예약·결제·매크로는 하지 않는다.
- 수집 주기는 과도하지 않게(사이트 부하 배려). 개발계정/단일계정 과다호출 주의.
- 내부 엔드포인트는 비공식 → 사이트 개편 시 유지보수 필요.
