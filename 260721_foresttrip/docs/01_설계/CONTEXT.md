# 숲나들e 알림 앱 (Forest Trip Notifier)

숲나들e(foresttrip.go.kr)에서 특정일에 예약/대기 가능한 자연휴양림·객실을 조회하고,
관심 객실이 예약/대기 가능해지면 알려주는 안드로이드 앱과 그 백엔드의 도메인 언어.

## Language

**휴양림 (Forest)**:
숲나들e에서 예약 가능한 자연휴양림 한 곳. 국립/공립/사립(insttTpCd 01/02/04)으로 구분되며 `insttId`로 식별된다. 예: `[공립](강화군)석모도자연휴양림`.
_Avoid_: 캠핑장, 리조트

**숙소상세 (Room)**:
휴양림 안의 개별 예약 단위(객실/시설). 사용자가 실제로 자고 싶어 하는 최소 단위이며 감시(Watch)의 대상이다. 예: `[숲속의집]2차숲속의집 5동 (화개산) (6인/48㎡)`. 로그인 세션이 있어야 조회된다(`fcfsRsrvtPssblGoodsDetls.do`).
_Avoid_: 객실상세, 시설상세, goods, 방

**예약가능 (Reservable)**:
해당 날짜에 선착순 즉시 예약이 열려 있는 상태. 검색 모드 `rsrvtWtngSctin=01`로 조회된다. 잡으면 바로 확정.
_Avoid_: 잔여, 가능

**대기가능 (Waitlistable)**:
즉시 예약은 불가하나 대기신청을 걸 수 있는 상태. 검색 모드 `rsrvtWtngSctin=02`(`fn_wtng_goSrch`)로만 조회된다. 대기신청은 숙박일 2일 전까지만 가능.
_Avoid_: 웨이팅, 대기예약

**대기순위 (Waitlist rank)**:
대기가능 상태의 숙소상세에서 현재 몇 번째 대기인지(예: `대기 2순위`). 대기 모드 조회 시 `.opt4`에 노출된다.
_Avoid_: 대기번호, 순번

**희망기간 (Desired range)**:
사용자가 특정 하루가 아니라 "이 기간 안 아무 날짜라도" 잡고 싶을 때 지정하는 날짜 구간. **박수(N박)는 고정**이고, 감시가 *연속 N박이 구간에 들어가는 모든 체크인 후보*를 훑어 하나라도 뜨면 알린다. 선택적 **요일 필터**(금·토만/평일만)를 건다.
_Avoid_: 기간, 날짜범위

**감시 (Watch)**:
사용자가 "이 대상을 이 날짜(또는 희망기간)로 잡고 싶다"고 등록해 두는 항목. 백엔드가 주기적으로(적응형 5–10분) 재조회하여 예약가능/대기가능으로 바뀌면 알림을 보낸다. 인원(nofpr)을 파라미터로 고정한다. 두 하위유형이 있다:
- **숙소상세 감시**: 특정 방 하나(goodsId/라벨)를 감시.
- **휴양림 감시**: 특정 휴양림에 "아무 방이나" 뜨면 알림(폴링 단위가 (휴양림,날짜)라 추가 부하 없음).
_Avoid_: 북마크, 관심등록, 알림설정, 구독

**선착순 (First-come)**:
평일 숙박처럼 실시간 잔여를 선착순으로 잡는 방식. `hldtList`로 날짜별 판정.
_Avoid_: 즉시예약(상태값 예약가능과 혼동 주의)

**추첨 (Lottery)**:
금·토·공휴일 전날 숙박처럼 응모 후 추첨으로 배정되는 방식. 선착순 조회에선 "예약불가"로 보이는 게 정상이라 사용자에게 별도 안내가 필요하다.
_Avoid_: 랜덤배정

**취소표 (Cancellation slot)**:
누군가 예약/대기를 취소해 잠깐 열리는 자리. 분 단위로 나타났다 사라지므로 감시·알림의 실제 표적이다. 취소표가 잘 뜨는 시간대를 "명당 시간"이라 부르고 패턴 학습의 대상이다.
_Avoid_: 취소자리, 빈자리

**전국 조회 (Nationwide search)**:
특정일(및 인원)로 전국에서 예약가능/대기가능한 휴양림·숙소상세를 한 번에 찾아 보여주는 조회. 감시 등록 이전의 탐색 단계.
_Avoid_: 전체검색, 스캔
