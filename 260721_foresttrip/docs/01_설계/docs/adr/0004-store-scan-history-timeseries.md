# 스캔 결과를 최신 스냅샷이 아니라 시계열 이력으로 저장한다

각 (휴양림, 날짜, 인원) 조회 결과를 **매 스캔 시각과 함께** 저장한다. 즉 "지금 상태"만
덮어쓰지 않고, 상태 전이(예약불가→예약가능, 대기 3순위→2순위, 열림→닫힘)를 타임스탬프가
붙은 **이벤트/시계열**로 남긴다.

**이유:** 플래그십 기능들이 과거 데이터에 의존한다.
- **취소표 패턴 학습·명당시간 예측**: 휴양림·시간대별로 취소표가 언제 뜨는지 = 열림 이벤트의
  시간 분포. 최신 스냅샷만으론 불가능.
- **대기 성공 확률 추정**: 대기순위가 시간에 따라 어떻게 줄어 예약가능/소멸로 갔는지의 궤적 필요.
- **"놓친 알림" 타임라인**: 취소표가 언제 떠서 얼마나 유지됐는지.

**결과:** SQLite 스키마는 현재상태 테이블 + append-only 이벤트 로그(또는 스캔 스냅샷 이력)로
구성한다. 개인 규모에선 SQLite로 충분하나, 이력은 무한 증가하므로 **보존기간/롤업 정책**
(예: 원시 이벤트 90일 후 집계로 다운샘플)을 둔다. 멀티유저 확장 시 Postgres/타임시리즈로 이전.
