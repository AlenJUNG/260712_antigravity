## 0. 한 줄 요약

숲나들e(foresttrip.go.kr)에서 **날짜 + (시/도 또는 전국)** 로 **예약가능/대기가능한 자연휴양림과 객실(시설)** 을 찾아,
표/웹페이지로 보여주는 서비스. 향후 안드로이드 앱의 백엔드. 현재 **PoC가 실동작**하며 전국 스캔 결과를 웹(Artifact)으로 발행까지 완료.

## 1. 목표 산출물(사용자 요구 컬럼)

시/도 · 휴양림명(`[공립](강화군)석모도자연휴양림`) · 개관연도 · 신축여부 · 숙박시설(숲속의집/휴양관/야영데크…) · 시설상세(`[휴양관]서어나무(2층) 5/5인실(31㎡)`) · 예약가능여부(예약가능/대기가능) · **대기순위(대기 2순위 등)**.

## 2. 현재 상태 (작동 확인됨)

- ✅ 시/도·휴양림 목록·휴양림 달력(휴무/추첨) : 순수 HTTP(무인증)
- ✅ 휴양림 단위 예약가능 조회 : Playwright로 NetFunnel 통과
- ✅ 국립 개관연도(47곳) 수집 + openYear/isNew
- ✅ 로그인(.env) → 객실 단위 상세(숙박시설/시설상세/예약·대기/대기순위)
- ✅ 로그인 세션 저장(auth.json) → **재로그인 없이 Claude가 직접 스캔**(병렬)
- ✅ 전국 스캔 결과 → 자체완결 HTML 대시보드(Artifact) 발행
  - 발행 URL: https://claude.ai/code/artifact/ff20df6c-51ff-4458-9c1d-7e6b5aea43b0

## 3. 아키텍처 & 접근 장벽

숲나들e는 조회 깊이에 따라 3단계 장벽이 있다:

| 장벽 | 대상 | 우회 |
|---|---|---|
| 없음 | 시/도·휴양림목록·달력 | fetch + 세션쿠키 + `_csrf` + `X-Ajax-call` 헤더 |
| **NetFunnel**(가상대기열) | 예약가능 조회 | Playwright로 페이지 로드 → `fn_top_goSearch()` 시 netfunnel_key 자동발급 |
| **로그인** | 객실 단위 상세·대기 | login.js(.env) 로그인 → 세션(auth.json) 저장 후 재사용 |

## 4. 내부 엔드포인트 (실측)

| 엔드포인트 | 게이트 | 반환 |
|---|---|---|
| `/rep/or/selectSiDoList.do` | - | 시/도 코드 JSON |
| `/rep/or/selectInsttListForSearch.do?srchSido=` | - | 시도별 휴양림 목록 JSON(insttId, insttTpCd 01/02/04) |
| `/rep/or/selectFcFsRcfrsFcltInfo.do?insttId=` | - | 휴양림 달력(정기휴무/주말추첨) |
| `/rep/or/fcfsRsrvtRcrfrDtlDetls.do` | NetFunnel | 지역 예약가능 휴양림 목록(HTML, `.rc_item`) |
| `/rep/or/sssn/fcfsRsrvtPssblGoodsDetls.do` | NetFunnel+로그인 | 객실 단위 상세(HTML, `.list_box`) |
| `/com/login` | - | 로그인(POST loginId/loginPwd/_csrf) |

**객실 DOM(.list_box)**: `.opt1`=[숙박시설]방이름, `.opt2`=인원/면적, `.opt3`=가격, `.opt4`=**대기순번(대기모드만, "2순위")**, `.btn_group .defBtn`의 `.txtRsrvt`(예약하기)/`.txtWtng`(대기신청) 중 `style.display!=none`=상태.

**예약(01) vs 대기(02) 모드**: 검색 시 `rsrvtWtngSctin=01`은 예약가능만 조회. 대기가능은 goods 페이지의 `fn_wtng_goSrch()`(=`rsrvtWtngSctin=02`)로 재조회해야 나온다. **대기신청은 숙박일 2일 전까지만 가능**(fn_wtng_goSrch 내 selectGoodsWtngAppliPssblDtYn.do 로 확인).

## 6. 명령어 (CLI)

```
node src/poc.js catalog [시도]                          카탈로그(무인증)
node src/poc.js search <시도> <입실> <퇴실> [인원]        휴양림 단위 예약가능(NetFunnel)
node src/poc.js rooms <시도> <insttId> <입실> <퇴실>      단일 휴양림 객실(로그인, 예약+대기)
node src/poc.js rooms-region <시도> <입실> <퇴실>         지역 전체 객실(로그인, 순차)
node src/poc.js rooms-nation <입실> <퇴실>                전국 객실(로그인, 순차)  ← 느림
node src/poc.js login-save                              로그인 후 세션을 auth.json 에 저장
node src/poc.js scan-nation <입실> <퇴실>                 ★auth.json 재사용+병렬 전국 스캔(권장)
node build-page.js                                     rooms-region.json → page.html
```

**권장 운영 흐름 (원격/빠름):**
1. (사용자, 세션 만료 시에만) `login-save.cmd` 더블클릭 → `auth.json` 생성.
2. (Claude) `CONCURRENCY=6 node src/poc.js scan-nation <입실> <퇴실>` — 세션 재사용, 비밀번호 없음, ~5–7분.
3. (Claude) `node build-page.js` → `artifacts/page.html` → Artifact 발행/갱신(같은 URL).

`.cmd` 실행파일: `run-rooms.cmd`(대관령 테스트), `run-nation.cmd`(전국 순차), `login-save.cmd`(세션저장). **모두 순수 ASCII 필수**(6절 오류이력 참고).

## 7. ⚠️ 오류 이력 & 해결 (반드시 숙지)

이 프로젝트에서 실제로 겪고 고친 문제들. 재발 방지용.

1. **`Cannot access 'TYPE' before initialization`**
   - 원인: `poc.js`에서 `const TYPE = {...}`를 top-level dispatch(`await runRoomsNation()`) **뒤**에 선언 → dispatch가 await되는 동안 TYPE는 TDZ(초기화 전).
   - 해결: `const TYPE`를 **dispatch 앞**(argv 파싱 직후)으로 이동.
   - 교훈: top-level await로 함수를 호출하는 모듈에선, 그 함수가 참조하는 const/let을 호출보다 위에 둘 것.

2. **Windows `.cmd` 실행 시 `'11'은(는) ... 명령이 아닙니다`**
   - 원인: 배치파일에 `chcp 65001`(UTF-8) + **한글 주석/문구**가 있으면, cmd가 UTF-8 전환 후 파일을 바이트 단위로 잘못 읽어 오프셋이 어긋남 → `node src\poc.js rooms 2 0111 ...` 줄을 `0111`의 `11`부터 읽어 `11`을 명령으로 실행.
   - 해결: **.cmd를 순수 ASCII로 작성, `chcp` 제거.** (한글 출력은 node가 run.log에 UTF-8로 저장 → iconv/UTF-8로 읽음.)
   - 검증: `grep -P "[^\x00-\x7F]" run-*.cmd` 로 비ASCII 없는지 확인.

3. **`!`(Claude Code 프리픽스) 명령이 실행 안 됨**
   - 증상: 사용자가 `!node src\poc.js ...` 입력해도 프로젝트 폴더에 아무 파일도 안 생김.
   - 추정: `!`가 프로젝트가 있는 원격 머신이 아니라 사용자 로컬(컨트롤러) 쪽에서 돌거나 미실행. 또한 상위 폴더에서 돌아 경로/.env 못 찾음.
   - 해결: 사용자는 **원격 데스크톱에서 .cmd 더블클릭**으로 실행(그 머신 위에서 돔). 이후엔 세션 재사용으로 Claude가 직접 실행.

4. **Headful 브라우저가 원격 화면에 안 보임(세션 격리)**
   - 증상: Claude가 headful 브라우저를 띄워 수동 로그인 시도 → 사용자가 창을 못 봐서 로그인 타임아웃.
   - 원인: Claude가 Bash로 띄운 프로세스가 사용자 원격 데스크톱과 다른 Windows 세션에서 실행됨.
   - 해결: headful 수동로그인 폐기. **headless .env 로그인(login-save.cmd, 사용자 더블클릭 1회) → 세션 저장 → 이후 Claude가 세션 재사용.**

5. **객실 상세가 로그인 화면 반환 (loginErrorCode -5)**
   - 원인: `fcfsRsrvtPssblGoodsDetls.do`는 로그인 필요.
   - 해결: `login.js`로 .env 로그인 후 같은 컨텍스트로 조회.

6. **예약 모드만 봐서 숙박(숲속의집/휴양관)이 안 나옴**
   - 원인: `rsrvtWtngSctin=01`(예약가능 목록)은 추첨/만석 숙박을 숨김. 주말(8/1)은 숙박이 추첨이라 캠핑만 나옴.
   - 해결: `includeWait`—goods 페이지에서 `fn_wtng_goSrch()` 호출해 대기(02) 재조회 → 숙박 대기물량 포착. 대기순번은 `.opt4`.

7. **속도(순차 ~35분) & 원격 반복실행 부담**
   - 해결: (a) 세션 저장(auth.json) → 재로그인 없이 Claude 실행(비밀번호 아님). (b) **워커풀 병렬(CONCURRENCY, 기본 6)** → ~5–7분. NetFunnel 동시 6요청 문제 없음(테스트 세션오류 0).

8. **저장된 goods.html을 Playwright로 로컬 검증 시 타임아웃**
   - 원인: 저장 HTML이 외부 스크립트/이미지를 로드하려다 `load`/`domcontentloaded` 안 뜸.
   - 해결: 파서 검증은 **Node에서 HTML 직접 정규식 파싱**으로 대체(라이브 페이지는 정상 로드되므로 extractRooms는 실동작).

## 8. 안전/자격증명 규칙 (중요)

- **비밀번호 입력(로그인)은 Claude가 대신 수행 불가**(안전 규칙, 사용자 허가해도 동일). 그래서 최초 로그인은 사용자가 `login-save.cmd`로 실행.
- **세션(auth.json) 재사용은 비밀번호 입력이 아니므로 Claude가 스캔 실행 가능.** 세션 만료 시에만 사용자가 login-save 재실행.
- `.env`(ID/PW)·`auth.json`·`*.log`는 `.gitignore`로 커밋 제외. Claude는 .env 값을 열람/로깅하지 않음.

## 9. 알려진 한계

- 개관연도/신축은 **국립 47곳만**(산림청 일반현황). 공립·사립·지방청 국유림(예: 진부령)은 `미상`.
- goods 목록이 휴양림당 **최대 ~10개**만 반환(페이지 크기). 방이 더 많으면 일부 누락 가능.
- 비공식 엔드포인트 → 사이트 개편 시 셀렉터/파라미터 깨질 수 있음.
- 세션 유효기간 불명(수 시간 추정). 만료 시 login-save 재실행.
- 조회/안내까지만. 자동예약·매크로 금지.
