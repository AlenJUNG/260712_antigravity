## 2. 프로 개발자들의 폴더 정리 원칙 및 사유

### 1) 루트 디렉토리의 단순화 (Clean Root Principle)
- **이유:** 프로젝트 루트는 사용자와 기여자가 가장 먼저 접하는 공간입니다. 설정 파일과 최상위 아키텍처 폴더(`src`, `docs`, `artifacts`)만 남기고 개별 작업 문서나 스크립트를 제거하여 **프로젝트 진입 시 인지 부하를 최소화**했습니다.

### 2) 역할별 디렉토리 격리 (Separation of Concerns)
- **`scripts/` 생성:** CLI 실행 스크립트(`.cmd`), 데이터 가공 스크립트(`.py`), 개별 수집기(`.js`) 등 7개 파일을 `02_구현/scripts/`로 이동시켰습니다.
- **이유:** 소스 코드(`src/`)와 일회성/수동 실행 도구(`scripts/`)가 섞여 있으면 코드 변경 추적(Git Diff)이나 빌드 시 파악이 어려워지므로 명확히 분리합니다.

### 3) 중복 제거 및 단일 진실 원천 (Single Source of Truth)
- **이유:** 루트와 `02_구현/` 양쪽에 다르게 남아있던 `전국휴양림정보.xlsx` 중 루트의 파일과 MS Office 잔여 파일(`~$전국휴양림정보.xlsx`)을 제거하고 `02_구현/` 내부로 일원화했습니다.

### 4) `OLD/` 디렉토리를 통한 데이터 격리 (Archiving Pattern)
- **이유:** 임시 실행 로그(`run-login.log`, `run.log`), 이전 중복 파일 등은 함부로 즉시 삭제하기보다 `OLD/` 디렉터리로 한데 격리 보관함으로써, 안전하게 히스토리를 보존하면서도 실용 공간을 정돈합니다.
