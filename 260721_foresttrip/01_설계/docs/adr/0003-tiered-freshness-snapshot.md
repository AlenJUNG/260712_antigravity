# 전국 조회는 스냅샷 + 온디맨드 실시간의 2티어로 제공한다

전국 특정일 조회를 다음 2티어로 서빙한다:
1. **스냅샷(즉시):** 백엔드 스케줄러가 주기적으로 스캔해 DB에 저장한 결과를 앱이 즉시 표시(몇 분 지연).
2. **온디맨드(실시간):** 사용자가 특정 휴양림을 탭하면 그 휴양림만 실시간 재조회(수초)로 확정.

배경 폴링은 전국 전체를 상시 갈지 않고 **사용자가 조회/감시한 (휴양림, 날짜)**로 한정한다.

**이유:** "예약가능 휴양림"은 휴양림 단위(로그인 불필요, 시도당 1요청)라 값싸지만, **"대기가능"은
숙소상세(방) 단위 개념**(`fn_wtng_goSrch`, 로그인, 휴양림마다 1요청)이라 전국 방 단위 스캔이
병렬로도 ~5–7분·고부하다. 따라서 "전국 대기가능"을 즉시 낼 방법은 원리상 없고, 스냅샷이
불가피하다. 매 검색 라이브 스캔은 UX(6분 대기)와 사이트 부하 모두에서 지속 불가능하다.

**결과:** 백엔드에 스케줄러 + 스냅샷 저장소(SQLite→서버 DB)가 필수 구성요소가 된다.
데이터는 "몇 분 지연"이 기본 계약이며, 사용자가 실제 행동(예약 이동) 직전엔 온디맨드 재조회로
최신을 확정한다.
