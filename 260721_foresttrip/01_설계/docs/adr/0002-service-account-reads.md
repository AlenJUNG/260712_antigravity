# 숙소상세 조회는 로그인 세션으로 하고, 자격증명은 백엔드에만 둔다

숙소상세(객실 단위)는 로그인 세션이 있어야 조회된다(`fcfsRsrvtPssblGoodsDetls.do`).

**개인 MVP("일단 나만 씀"):**
- 앱에 숲나들e 로그인 화면을 두고 **소유자가 자기 ID/PW를 1회 입력**한다.
- 백엔드가 그 자격증명으로 로그인 → **재사용 세션**(auth.json 상당) 확보 → 모든 폴링/조회에 사용.
- 자격증명은 **백엔드에만 암호화 저장**하고, 세션 만료 시 **headless 자동 재로그인**에 쓴다.
  앱에는 **백엔드 세션토큰만** 남기고 숲나들e 비밀번호를 저장하지 않는다.
- 서비스계정 = 소유자 본인 계정. 즉 "서비스계정 vs 사용자계정" 구분이 이 단계에선 합쳐진다.

**확장(멀티유저) 단계:**
- 숙소상세 조회는 *아무* 유효 로그인 세션이면 되고, 앱이 자동예약을 하지 않으므로
  ([[0001-notify-and-handoff-no-autobook]]) 타 사용자별 비밀번호는 원리상 불필요하다.
  이때는 소수 **서비스계정 풀** + 레이트리밋으로 부하·차단을 분산한다. "서비스계정으로
  불특정 타인 대량 조회"는 ToS 회색지대이므로 이 단계에서 재검토한다.

**이유:** 실제 로그인/폴링을 백엔드(Playwright)가 수행하므로 자격증명이 백엔드에 있어야
자동 재로그인이 가능하고(폴링이 조용히 죽는 것 방지), 앱에 비번을 남기지 않아 기기 분실 시
노출면이 최소화된다. 자격증명 평문 로깅은 금지하고 암호화 저장한다.

**주의:** 비밀번호 5회 오류 시 계정 잠금 → 자동 재로그인은 **실패 시 재시도 금지**(백오프 후
소유자에게 "재로그인 필요" 알림). Claude(어시스턴트)는 비밀번호를 직접 입력하지 않는다 —
입력 주체는 앱 사용자, 자동 재로그인 주체는 코드다.
