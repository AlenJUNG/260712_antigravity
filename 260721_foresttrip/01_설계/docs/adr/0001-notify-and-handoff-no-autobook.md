# 앱은 예약하지 않는다 — 알림 + 딥링크 핸드오프만

관심 숙소상세가 예약가능/대기가능으로 바뀌면 앱은 **푸시 알림**을 보내고, 사용자를
숲나들e **공식 예약 페이지로 딥링크**시키는 데서 멈춘다. 앱이 자동/반자동으로 예약이나
대기신청을 대신 누르지 않는다.

**이유:** 자동예약·매크로는 이 프로젝트의 하드룰 위반이며 계정정지·법적 리스크가 크다.
반면 "알림 → 원탭 핸드오프 → 사용자가 직접 확정"은 합법성(조회·안내 범위)을 지키면서도
실사용 속도 손실이 거의 없다. 사용자의 *"기다렸다가 바로 예약되면 더 좋다"* 요구는
**자동예약이 아니라, 즉시예약(예약가능)으로 열린 건을 대기건보다 최우선 등급으로 강조 알림**
하는 것으로 충족한다.

**결과:** 앱은 사용자의 숲나들e 비밀번호로 "예약 행위"를 할 필요가 없어진다 → 인증 모델이
크게 단순해진다([[0002-service-account-reads]] 참고).
