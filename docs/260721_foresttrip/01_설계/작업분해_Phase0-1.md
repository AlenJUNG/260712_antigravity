      combined ~20 truncate(rooms-region.json 20-상한 확증). → 확정: **각 모드 전 페이지 순회**
      (`.paging_count`에서 N 읽고 `fn_goPage('2')…('N')` 구동, `goodsId` dedup 누적). 내부 AJAX는
      `/rep/or/sssn/innerFcfsRsrvtPssblGoodsDetls.do`. (설계서 §6-bis)
      → Phase 1 구현 태스크는 §1.1로 편입.
- [~] `[R]` 세션 유효기간·자동 재로그인 신뢰성 실측(만료 감지 신호, 백오프 값). — **부분 확정
      (유일하게 남은 조사 항목)**
      - 실측(2026-07-23): `auth.json`(2026-07-21 생성)이 **31.7시간 뒤 만료**됨을 확인.
        `JSESSIONID`가 세션쿠키이므로 서버 측 만료다. **정확한 TTL은 미측정**(31.7h는 상한).
      - 유력 가설: 일반적인 J2EE **비활동 기준 타임아웃**(수십 분). 사실이라면 3~10분 주기로
        도는 폴링 자체가 세션을 살려두므로, 만료는 **장시간 유휴 뒤**에만 문제가 된다.
      - 구현은 이미 "goods가 로그인화면 반환 = 세션 무효화 → 다음 요청에서 1회 재로그인,
        실패 시 무재시도 + 소유자 푸시"로 대응한다. **남은 확인은 24h 소크 테스트로 흡수한다.**
- [x] `[R]` **레이트버짓 안전값(ADR-0006 `R`)** — **확정: `R = 20 req/min`, `N_max = 20`**
      - **차단될 때까지 올려보는 실측은 하지 않기로 했다.** 소유자 실계정 하나로 소프트밴·잠금을
        유발하는 실험이라 ADR-0006이 막으려는 사고를 자초하는 꼴이다.
      - 대신 기존 무사고 실적을 근거로 삼았다: `scan-nation` CONCURRENCY=6 전국 스캔 5–7분,
        **세션오류 0** → 약 **65–90 req/min 버스트 통과**. 그 1/4인 20을 지속 상한으로 채택.
      - 폴당 비용도 정정(`pagesTraversed + 1`). 상향은 24h 게이트 통과 후에만.
- [x] `[R]` **min-nofpr 폴 병합(ADR-0006 §4)** — **성립 확인 + 적용 완료**
      - 강원 8/13: 인원 2/4/6/8에 예약가능 9/9/7/5곳, 수도권 8/26: 2/6/10에 16/16/9곳.
        **단조 감소 + 낮은 인원에 없던 항목 등장 0건** → `nofpr=2` 결과가 상위집합.
      - 총 객실수는 인원과 무관하게 일정 → 인원은 **방 수용인원 필터**로만 작용.
      - `poll_target`을 (휴양림,날짜,박수)로 묶어 최소 인원 1회 폴링 + `capacity` 다운스트림 필터로
        구현. 파싱 불가 방은 fail-open(과알림 ≫ 누락).
      - ⚠ 검증은 **휴양림 단위**(무로그인)에서 수행. 방 단위 재확인은 로그인 후 필요하나,
        틀려도 실패 양상은 **과알림이지 누락이 아니다**(항상 상위집합을 폴링).
