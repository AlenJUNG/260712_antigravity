# 숲나들e 자연휴양림 예약가능 조회 — 코드 설계서

> 날짜와 시/도를 지정하면 숲나들e(foresttrip.go.kr)에서 예약(또는 대기) 가능한
> 전국 자연휴양림과 시설을 찾아 표로 제공하는 서비스. 향후 안드로이드 앱의 백엔드.

- 문서 버전: 0.3 (2026-07-21)
- 대상 사이트: https://www.foresttrip.go.kr
- 코드 위치: `260721_foresttrip/`

---

## 1. 목적과 범위

| 구분 | 내용 |
|---|---|
| 목적 | "희망 날짜 + 시/도 → 예약/대기 가능한 휴양림·시설" 조회 |
| 커버리지 | 국립 + 공립 + 사립 (숲나들e에서 예약 가능한 전국 전체) |
| 산출물 | 시/도 · 휴양림명 · 개관연도 · 신축여부 · 숙박시설 · 시설상세 · 예약가능여부 |
| 비범위 | 자동 예약/결제/매크로 (금지). 본 서비스는 **조회·안내**까지만 |

## 2. 전체 아키텍처

```
┌────────────────────────────────────────────────────────────────┐
│  수집 계층 (Node.js + Playwright)                                │
│                                                                  │
│  ① 카탈로그/메타      : 순수 HTTP(fetch)  — 무인증               │
│     - 시/도 목록, 시도별 휴양림 목록, 휴양림 달력(휴무/추첨)     │
│  ② 개장연도            : 산림청 일반현황 스크래핑 → 정적 JSON    │
│  ③ 휴양림 단위 가용성  : Playwright(NetFunnel 통과) — 무인증     │
│  ④ 객실 단위 가용성    : Playwright(NetFunnel + 로그인) — 인증   │
│                                                                  │
│                     ↓ 정규화 · openYear/isNew 부착               │
│                  [ 캐시(현재 인메모리) / 향후 DB ]               │
└───────────────────────────────┬────────────────────────────────┘
                                 │
                     REST API 서버 (server.js)
                     GET /available?sido=&date=
                                 │
                                 ↓
              웹 프론트 / (향후) 안드로이드 앱  ← REST 클라이언트
```

핵심 원칙: **Playwright 스크래퍼는 서버에서만 실행**된다(앱 내부 실행 불가).
앱은 REST API를 호출하는 얇은 클라이언트이므로 앱 프레임워크(Kotlin/Flutter 등)는
백엔드와 독립적으로 선택한다.

## 3. 데이터 소스와 접근 방식

숲나들e는 조회 깊이에 따라 3단계 접근 장벽이 있다.

| 접근 장벽 | 대상 | 우회 방법 |
|---|---|---|
| 없음 (공개 HTTP) | 시/도·휴양림 목록, 휴양림 달력 | `fetch` + 세션쿠키 + `_csrf` + `X-Ajax-call` 헤더 |
| **NetFunnel** (가상대기열) | 휴양림 단위 예약가능 목록 | Playwright로 실제 페이지 로드 → `netfunnel_key` 정상 발급 |
| **NetFunnel + 로그인** | 객실 단위 상세(숙박시설·시설상세·대기) | 위 + 숲나들e 계정 로그인 세션 |

> 공공데이터포털 API는 국립만 커버 + "예약현황(확정/대기/취소)"이라 잔여 계산에
> 부적합. 전국·실시간 가용성은 사이트 조회만이 충족한다. (조사보고서.md 참고)

## 5. 모듈 설계

```
src/
  constants.js            BASE URL, UA, 시/도 코드맵, 엔드포인트 경로
  session.js              공개 HTTP 세션(쿠키+_csrf) + ajaxGet 공통 호출기
  catalog.js              시/도·휴양림 목록·휴양림 달력 (무인증)
  browser.js              Playwright 브라우저 싱글턴 (요청마다 컨텍스트만 생성)
  availability.js         휴양림 단위 예약가능 조회 (NetFunnel 통과) + openYear 부착
  login.js                숲나들e 로그인 → 로그인된 컨텍스트 (env 자격증명)
  goods.js                객실 단위 상세 조회 (로그인 컨텍스트 재사용)
  collectNationalYears.js 국립 개장연도 수집기 → data/national-open-years.json
  openYear.js             이름 매칭으로 openYear/isNew 부착 (별칭 보정 포함)
  cache.js                인메모리 TTL 캐시 (향후 DB 교체)
  server.js               REST API 서버 (/health /sido /available)
  poc.js                  CLI 진입점 (catalog/search/rooms)
data/
  national-open-years.json  국립 47곳 지정/조성/개장 연도 (수집 산출물)
```

각 모듈의 단일 책임:
- **session/catalog**: 로그인·NetFunnel 없이 얻을 수 있는 모든 정형 데이터.
- **browser**: Chromium 1회 기동 후 재사용(요청마다 재기동 방지). `HEADLESS=1` 지원.
- **availability**: 지역+날짜 → `.rc_item` 파싱 → 휴양림별 예약가능/불가 + 객실수.
- **login/goods**: 로그인 세션이 필요한 객실 단위. login이 컨텍스트를 만들고 goods가 재사용.
- **openYear**: 국립 개장연도 부착(공립·사립은 데이터 없음 → null).

## 6. 데이터 흐름 (시퀀스)

### 6.1 휴양림 단위 (무인증) — `GET /available`
```
클라이언트 → server.js
  캐시 HIT? → 반환(0ms)
  MISS →
    availability.searchRegionAvailability(sido, checkin, checkout)
      browser(공유) → newContext → page
      goto fcfsRsrvtMain.do           (netfunnel.js 로드)
      필드 주입(srchInsttArcd/날짜/인원)
      fn_top_goSearch()               (NetFunnel 키 발급 + form GET submit)
      → fcfsRsrvtRcrfrDtlDetls.do 결과 페이지
      → innerFcfsRcrfrDtlDetls.do 가 .rc_item 목록 채움
      parse .rc_item → [{name,type,status,available,rooms}]
      map(withOpenYear) → openYear/isNew 부착
    캐시 SET → 반환
```

### 6.2 객실 단위 (로그인) — `rooms` (현재 CLI)
```
login.createLoggedInContext(FOREST_ID, FOREST_PW)
  goto /com/login.do → fill #mmberId/#gnrlMmberPssrd → fn_goLogin() → 성공 판정
  → 로그인된 컨텍스트 반환
goods.searchForestGoods(context, sido, insttId, checkin, checkout)
  같은(로그인) 컨텍스트로 fcfsRsrvtMain → 특정 휴양림 지정 → fn_top_goSearch()
  → fcfsRsrvtPssblGoodsDetls.do (객실 상세, 로그인 세션 필요)
  parse 객실 → [{숙박시설, 시설상세, 예약/대기}]   ← 파서 확정 대기(§10)
```

```

```

## 8. 도메인 규칙

- **선착순 vs 추첨**: 평일 숙박은 선착순(실시간 잔여), 금·토·공휴일 전날은 추첨.
  `selectFcFsRcfrsFcltInfo.do` 의 `hldtList`(정기휴무일/주말 추첨/공휴일)로 구분.
  → 토요일 등 추첨일은 선착순 조회에서 `예약불가`로 나오는 게 정상.
- **대기(waitlist)**: 객실 단위에서만 "대기가능"으로 구분됨(로그인 필요).
- **개관연도/신축(openYear/isNew)**: 국립 47곳만 신뢰 가능(산림청 일반현황).
  공립·사립·지방청 국유림은 정형 소스 없음 → `null`. `isNew = thisYear - openYear ≤ N`
  (`NEW_WITHIN_YEARS`, 기본 3).
- **휴양림 구분(type)**: 예약 결과의 `[국립]/[공립]/[사립]` 접두 = insttTpCd 01/02/04.

## 9. 보안 / 자격증명 처리

- 로그인 자격증명은 **프로젝트 루트 `.env`** 의 `ID` / `PW`(FOREST_ 접두도 지원)로만 주입.
  `poc.js`/`server.js` 가 `process.loadEnvFile()` 로 자동 로드.
- `.env` 는 `.gitignore` 에 포함 → **절대 커밋되지 않음**.
- 코드는 자격증명 값을 저장/로깅하지 않는다.
- 로그인 폼: `POST /com/login`, 필드 `loginId`/`loginPwd`, **캡차 없음, 클라이언트
  암호화 없음(HTTPS 평문), 비밀번호 5회 오류 시 계정 잠금** → 재시도 금지.

## 10. 알려진 한계 & 확정 필요 항목

1. **객실 파서 미확정**: `goods.js` 의 객실 파서는 잠정본. 로그인 실행분의
   `artifacts/goods.html` DOM으로 `숙박시설`/`시설상세`/`예약·대기` 셀렉터 확정 필요.
2. **비정형 커버리지**: 공립·사립 개관연도 없음. 지방청 국유림(예: 진부령)은
   국립 표에도 없어 `openYear:null`.
3. **비공식 엔드포인트**: 사이트 개편 시 셀렉터/파라미터가 깨질 수 있음(유지보수 필요).
4. **성능**: NetFunnel 통과는 요청당 ~6–10초 → 캐시 필수. 대량 조회는 수집기+DB로 전환.

## 12. 운영 워크플로 (세션 재사용 · 병렬 · 웹 발행)

원격 환경 + 속도를 위해 다음 흐름으로 운영한다.

```
① (사용자, 세션 만료 시만)  login-save.cmd 더블클릭
     → login.js(.env) headless 로그인 → ctx.storageState → auth.json (세션쿠키)
② (Claude, 비밀번호 없음)   scan-nation <입실> <퇴실>   (CONCURRENCY=6)
     → auth.json 재사용 컨텍스트 + 워커풀 병렬
     → 각 휴양림: 예약(01) + 대기(02) 조회 → openYear 병합
     → artifacts/rooms-region.json (중간 저장)
③ (Claude)  build-page.js → artifacts/page.html → Artifact 발행/갱신(같은 URL)
```

- **왜 Claude가 ②를 실행 가능한가:** 세션쿠키 재사용은 "비밀번호 입력"이 아니므로 안전규칙에 걸리지 않는다. 최초 로그인(①)만 사용자가 수행.
- **성능:** 순차 ~35분 → 병렬 6 ~5–7분(NetFunnel 동시요청 문제 없음, 실측 세션오류 0).
- **원격 조회:** 결과는 self-contained HTML(Artifact)로 발행 → 링크로 어디서든 확인. 데이터는 페이지에 임베드(외부 fetch 없음, CSP 준수).
- 상세 인수인계·오류이력: `인수인계.md`.
