# 숲나들e 자연휴양림 예약가능 조회 서비스 사용 가이드

본 서비스는 전국 자연휴양림의 실시간 예약 및 대기 현황을 모니터링하고, 지정한 조건이 활성화될 때 푸시 알림을 제공하는 백엔드 시스템입니다. 

사용자가 숲나들e 사이트를 수동으로 반복해서 방문하지 않도록 자동화된 모니터링 스케줄러와 SQLite 데이터베이스, 모바일/웹 앱 통신용 REST API를 제공합니다.

## 2. 시작하기 (설치 및 설정)

### Step 1. Node.js 종속성 설치
명령 프롬프트(CMD) 또는 터미널을 열고 `02_구현` 디렉토리로 이동한 후 npm 패키지를 설치합니다.
```bash
cd D:\git\260712_antigravity\260721_foresttrip\02_구현
npm install
```

### Step 2. 환경변수 설정 (`.env`)
`02_구현/.env` 파일을 메모장이나 에디터로 열어 필요한 값을 설정합니다.
```ini
PORT=3000
APP_TOKEN=your_secure_bearer_token_here # 앱과 서버 간 통신에 쓰일 보안 인증 토큰
ENCRYPTION_KEY=your_secure_crypto_key   # DB 내 숲나들e 비밀번호 암호화용 마스터 키
```
> [!IMPORTANT]
> - `APP_TOKEN`은 외부 통신 보안용 토큰입니다. 모바일 앱이나 웹 프론트엔드가 백엔드 API를 호출할 때 헤더에 `Authorization: Bearer <APP_TOKEN>` 형태로 포함해야 합니다.
> - `ENCRYPTION_KEY`는 내부 SQLite DB에 사용자의 숲나들e 비밀번호를 암호화하여 보존할 때 사용하는 고유 키입니다.

## 3. 주요 스크립트 실행 가이드

### ① 전국 휴양림 정보 엑셀 생성 및 갱신
전국 186개 자연휴양림의 기본 정보, 개관연도, 신축 여부(2026년 기준 6년 이내), 계곡 유무, 물놀이장 유무 등을 분석하여 보기 좋게 서식이 적용된 엑셀 파일을 생성합니다.
```bash
python make_excel.py
```
- **결과물**: `전국휴양림정보.xlsx` 파일이 생성되거나 갱신됩니다.
- **캐시**: 계곡/물놀이 시설 정보 수집은 `scrape_forest_features_daum.py`를 통해 웹 검색 데이터와 대조 수집되며, 최초 수집 이후에는 `data/forest-features.json` 캐시를 읽어 지연 없이 즉시 반영됩니다.

### ② 백엔드 API 서버 및 폴링 스케줄러 기동
스마트폰 앱과의 통신을 중계하는 API 서버와 백그라운드에서 예약 현황을 실시간으로 감시하는 스케줄러를 한꺼번에 구동합니다.
```bash
npm run serve
```
- **서버 기동 시 수행 작업**:
  1. `data/forest.db` SQLite 데이터베이스가 없는 경우 자동 생성하고 스키마를 마이그레이션합니다.
  2. 186개 휴양림의 정적 메타데이터를 DB의 `forest` 테이블로 자동 파퓰레이트합니다.
  3. 백그라운드에서 적응형 폴링 타이머가 5초마다 동작하며 감시 대상 큐를 처리합니다.

## 4. REST API 활용 안내

서버가 켜진 상태에서 앱은 다음 API 엔드포인트들을 호출할 수 있습니다. 
*(모든 API 호출 시 HTTP 헤더에 `Authorization: Bearer <APP_TOKEN>` 추가 필수)*

### 1) 헬스 체크
- **URL**: `GET /v1/health`
- **설명**: API 서버의 생존 여부와 현재 숲나들e 세션 유효 상태를 반환합니다.

### 2) 숲나들e 소유자 로그인 자격증명 등록
- **URL**: `POST /v1/auth/forest-login`
- **Body**:
  ```json
  {
    "loginId": "your_foresttrip_id",
    "loginPwd": "your_foresttrip_password"
  }
  ```
- **설명**: 숲나들e ID와 비밀번호를 1회 입력받아 암호화(AES-256-GCM) 후 DB에 저장하고, 즉시 Playwright 브라우저를 통해 첫 로그인 검증을 시도하여 성공하면 브라우저의 로그인 토큰(Cookie, LocalStorage) 세션을 캐시합니다. 
- *비밀번호 5회 오류 시 계정이 잠기므로, 최초 1회 성공 검증 후에만 저장이 완료되도록 방어 로직이 적용되어 있습니다.*

### 3) 실시간 전국/지역 스냅샷 검색
- **URL**: `GET /v1/search?date=YYYYMMDD&nights=1&nofpr=2&sido=시도코드`
- **설명**: 휴양림 단위의 예약 가능 여부 스냅샷을 즉시 로드합니다. 
  - `availSnapshotAt`: 값싼 시도별 가용성 타임스탬프 (즉시 반영)
  - `waitSnapshotAt`: 비싼 객실 단위 대기 현황 타임스탬프 (현재 감시 중인 타깃만 실시간 갱신되어 제공되며, 감시 외 대상은 null로 반환되어 탭할 때 실시간 조회되도록 유도)

### 4) 실시간 온디맨드 객실 조회
- **URL**: `GET /v1/forests/{insttId}/rooms?date=YYYYMMDD&nights=1&nofpr=2`
- **설명**: 특정 휴양림의 상세 방 목록을 실시간으로 긁어옵니다.
  - **페이지네이션 순회 보완**: 예약(01) 및 대기(02) 상태의 모든 객실을 가져오기 위해 Playwright 브라우저가 AJAX 페이징(`fn_goPage`)을 끝까지 순회하여 한 개의 방도 누락되지 않도록 수집하여 반환합니다.

### 5) 감시 대상 등록 및 모니터링
- **URL**: `POST /v1/watches`
- **Body**:
  ```json
  {
    "deviceId": "device-uuid",
    "type": "room", // forest(아무 방이나) 또는 room(특정 객실)
    "insttId": "0111",
    "goodsId": "G12345",
    "date": "20260819",
    "nights": 1,
    "nofpr": 2,
    "notifyGrades": ["예약가능", "대기가능"]
  }
  ```
- **설명**: 감시 대상 큐에 등록되면, 스케줄러가 타깃의 D-day 임박 여부, 피크 시간대(09시~18시) 여부를 계산하여 폴링 주기를 **최소 3분에서 최대 10분**까지 동적으로 제어하여 실시간 체크합니다.
- 상태가 `닫힘 -> 열림(예약가능/대기가능)`으로 전환되거나 대기 순번이 변경되는 경우 즉시 `data/pushes.log`에 로그를 찍고 푸시 알림(FCM Stub)을 발송합니다.
- 대기 알림의 경우 **D-2일(대기 마감 시점)**에 감시를 자동 종료하고 정리를 자동 수행합니다.

## 5. 관리 및 모니터링 팁

- **Slow / Softban 방지**: 
  - 백그라운드 수집 실행 시 숲나들e 로그인 세션이 끊기거나 차단당할 경우 스캔 기록(`scan` 테이블)에 `ok = 0`으로 기록됩니다. 
  - 스케줄러는 연속적으로 스캔에 실패할 경우 동작을 멈추고 자격증명 재로그인(`needsRelogin = 1`)을 유도하도록 되어 있습니다.
- **로그 확인**:
  - `data/pushes.log`: 발송된 푸시 알림 로그 및 딥링크 정보 확인
  - `02_구현/run.log`: CLI 수집 테스트 시의 콘솔 로그
