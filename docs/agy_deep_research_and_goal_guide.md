# 📖 Google Antigravity CLI (agy) 딥 리서치 및 Goal 모드 활용 매뉴얼

Google Antigravity CLI(`agy`)는 단순한 코드 완성을 넘어, 다단계 웹 브라우징과 비동기 백그라운드 작업을 실행할 수 있는 에이전트 인프라를 갖추고 있습니다. 이 매뉴얼은 `agy`에서 **자연어 프롬프트 기반의 딥 리서치(Deep Research)**를 실행하는 방법과 **목표 지향형 장기 실행 모드인 `/goal`**을 결합하여 고도화된 조사/코딩 결과물을 얻는 방법을 상세히 설명합니다.

---

## 1. 🔍 자연어로 딥 리서치(Deep Research) 지시하기

`agy`는 TUI 슬래시 커맨드로 `/research`를 명시적으로 가지고 있지 않지만, 사용자가 **자연어로 지시하면 백그라운드에서 전용 `research` 서브에이전트나 브라우징 도구들을 연쇄 호출**하여 딥 리서치를 자동으로 가동합니다.

### 💡 딥 리서치 작동 원리
1.  **프롬프트 분석**: 사용자가 "딥 리서치", "심층 조사", "웹 검색 교차 검증" 등의 핵심 키워드를 입력하면, 에이전트는 일반 대화 모드에서 리서치 전용 실행 흐름으로 전환합니다.
2.  **`research` 서브에이전트 가동**: 메인 에이전트가 리서치 전용 서브에이전트를 생성하거나 직접 `search_web`, `read_url_content` 등의 도구를 반복 호출합니다.
3.  **교차 검증 및 보고서 작성**: 최소 3개 이상의 소스를 분석하여 상충되는 정보를 걸러내고, 각 주장마다 신뢰할 수 있는 출처(Citations)를 매칭한 뒤 마크다운 형태의 결과물로 종합합니다.

### 📝 실전 리서치 지시 프롬프트 템플릿
*   **기술 스펙 및 API 조사**
    > *"**[기술/라이브러리 이름]**의 최신 스펙과 변경점에 대해 관련 테크 블로그 및 공식 문서를 **딥 리서치**해줘. 특히 이전 버전과의 차이점과 마이그레이션 시 주의사항을 포함해줘."*
*   **비교 분석 및 툴 추천**
    > *"**[A 도구]**와 **[B 도구]**를 비교하는 **심층 리서치**를 수행해줘. 각각의 비용(라이선스), 성능 메트릭, 실제 유저들의 사용성 평가를 취합하고, 우리 프로젝트(개발 스택: **[스택]**)에 어떤 것이 더 적합한지 추천안과 타당성 보고서를 작성해줘."*
*   **트러블슈팅 및 레퍼런스 발굴**
    > *"현재 발생한 에러 **[에러 메시지/상황]**에 대해 동일한 이슈를 겪은 개발자들의 해결 사례를 GitHub Issue 및 StackOverflow에서 **교차 검증하여 딥 리서치**해줘. 가장 신뢰성 높은 해결책 3가지와 예시 코드를 정리해줘."*

## 2. 🎯 `/goal` 커맨드로 딥 리서치 강제 및 자동화하기

`/goal`은 에이전트에게 1회성 답변을 요구하는 것이 아니라, **지정한 최종 목표(Goal)가 완벽하게 성취될 때까지 스스로 문제를 해결하고 도구를 연속 실행하도록 만드는 지속형(Loop) 모드**입니다. 딥 리서치와 `/goal`을 결합하면, 에이전트가 완벽한 보고서 파일을 생성하거나 테스트를 통과할 때까지 멈추지 않고 스스로 작업을 수행합니다.

### 🛠️ `/goal` 사용 단계
1.  **TUI 진입**: 터미널에서 `agy`를 실행합니다.
2.  **명령어 입력**: 대화창에 `/goal`을 입력하고 엔터를 누릅니다.
3.  **목표 작성**: 에이전트가 목표를 물어보면, **[목표 대상] + [조사 및 검증 기준] + [최종 산출물 경로 및 포맷]**을 구체적으로 전달합니다.

### 📝 `/goal` 리서치용 실전 템플릿
*   **가장 권장하는 목표 작성 양식:**
    > *"**[목표]**: **[조사할 주제]**에 대한 딥 리서치를 수행하라.  
    > **[검증 조건]**: 반드시 공식 문서와 최신 GitHub 저장소 레퍼런스 3개 이상을 교차 검증할 것.  
    > **[최종 산출물]**: 분석 리포트를 `./docs/[파일명].md`로 저장하고, 주요 개념 요약 테이블과 바로 적용 가능한 코드 스니펫을 포함할 것."*

### 🔄 실행 화면 예시
```text
User > /goal
Agent > 어떤 목표를 달성하고 싶으신가요? 최대한 구체적으로 작성해 주세요.
User > 목표: LivePortrait와 타 입모양 동기화(Lipsync) AI 기술들을 분석해줘. 해외 논문 및 깃허브 트렌드를 딥 리서치하여 비교 보고서를 `./docs/lipsync_comparison_report.md` 파일로 작성해줘.
Agent > 목표를 접수했습니다. 백그라운드 리서치를 시작합니다... (루프 가동)
```

## 3. 📊 백그라운드 작업 모니터링 및 제어

딥 리서치와 Goal 모드는 백그라운드에서 비동기로 실행되므로, 메인 화면을 가로막지 않고 다른 작업을 할 수 있습니다. 이때 아래 명령어를 통해 현재 작업을 제어할 수 있습니다.

### 1) 주요 상태 제어 명령어
*   **`/agents` 또는 `/tasks`**: 현재 가동 중인 백그라운드 리서치 서브에이전트 및 태스크 목록을 확인합니다.
*   **에이전트 상세 정보 확인**: `/agents` 패널에서 타겟 서브에이전트를 선택하면 에이전트의 내부 생각(Reasoning Chain), 현재 실행 중인 도구(웹 브라우저 등), 실시간 로그를 상세히 모니터링할 수 있습니다.
*   **`/exit` 또는 `Ctrl+D`**: 작업을 유지한 채 TUI 콘솔에서 빠져나올 수 있습니다. (백그라운드 에이전트는 계속 동작)

### 2) 고급 실행 옵션
*   **자동 권한 건너뛰기 (`--dangerously-skip-permissions`)**
    리서치 도중 에이전트가 임시 파일을 쓰거나 스크립트를 실행할 때마다 사용자에게 뜨는 승인(Approve) 확인창을 생략하고 완전 자동화하려면, `agy` 실행 시 이 플래그를 추가합니다.
    ```bash
    agy --dangerously-skip-permissions
    ```
    *※ 신뢰할 수 있는 네트워크 환경과 샌드박스 설정 내에서 사용할 것을 권장합니다.*

## 4. 🚀 딥 리서치 효율을 극대화하는 꿀팁

1.  **컨텍스트 유지하기 (`--continue` 또는 `-c`)**
    어제 지시해 둔 딥 리서치 세션이나 Goal 상태를 이어서 진행하려면 세션을 새로 열지 말고 복구 플래그를 활용하세요.
    ```bash
    agy --continue
    ```
2.  **룰셋 지정으로 리서치 규격 통일하기 (`/learn` 활용)**
    리서치 보고서를 작성할 때 선호하는 양식(예: "모든 기술 조사는 장단점 표를 반드시 포함할 것", "문서는 항상 `./docs/` 아래에 저장할 것")이 있다면, 대화 도중 `/learn` 명령어를 통해 에이전트의 지식 저장소(Knowledge Base)에 영구 각인시킬 수 있습니다.
