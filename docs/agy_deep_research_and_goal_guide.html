Deep Research & Goal Mode

Google Antigravity CLI(agy)에서 최상의 개발 분석 및 자율 조사 성과를 도출해내는 심층 제어 매뉴얼

자연어 리서치 지시 방법

agy 에이전트는 별도의 셸 명령어 대신, 자연어 질문에 포함된 '리서치', '검색', '스펙 조사'와 같은 맥락을 스스로 판단하여 다단계 웹 검색 및 문서 조합 태스크를 시작합니다.

단순 검색에 그치지 않고 여러 블로그나 문서를 서로 교차 분석하여 정확한 결론을 도출합니다.

에이전트는 내부적으로 research라는 특수한 읽기 전용 서브에이전트 인스턴스를 생성해 웹에 접근하므로 안전성이 보장됩니다.

실전 명령어 형태

TUI 콘솔 내 혹은 외부 터미널 셸(Shell)에서 각각 다음과 같이 지시할 수 있습니다.

1
TUI 대화형 모드

agy 프롬프트 창에 바로 리서치 대상을 설명합니다.

"LivePortrait의 요구 리소스를 조사해줘."
2
터미널 비대화형 모드 (Headless)

터미널에서 1회성 출력을 즉시 받고 싶을 때 사용합니다.

agy -p "Next.js 15 App Router 변경사항 조사"

/goal 명령어의 이해

/goal 명령어는 에이전트에게 '목표가 완수되어 모든 테스트와 검증을 끝마칠 때까지 포기하지 말고 반복 루프를 가동하라'고 명시하는 궁극적인 자동화 도구입니다.

특히 조사 항목이 방대하고 최종적으로 마크다운 문서 작성이나 스크립트 빌드까지 완료해야 할 때 베스트입니다.

1
목표 설정 구체화

단순 질문보다는 '목표 + 제약사항 + 최종 산출물 규격'을 한꺼번에 적어야 최고의 효율이 납니다.

2
실행 관리

루프가 가동되는 동안 메인 창을 나가거나 다른 터미널에서 작업해도 비동기 백그라운드 태스크로 에이전트가 알아서 동작합니다.

골 모드 실전 사용 예시

TUI 내에서 실행하는 시나리오와 최종 결과 리포트를 작성하게 하는 커맨드 구조입니다.

Interactive Session 
User > /goal
Agent > 무엇을 달성하고 싶으신가요?
User > 목표: LivePortrait와 live2d의 구동 메커니즘을 심도 있게 비교 분석해줘. 두 기술의 장단점 분석표를 포함한 보고서를 작성해 `./docs/comparison.md`에 저장해.
